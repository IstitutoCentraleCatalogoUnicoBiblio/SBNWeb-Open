Descrizione

Il Caso d’Uso consente al Bibliotecario di personalizzare i default applicativi all'area selezionata, in questo caso gestione bibliografica.
Ogni area di default a sua volta viene suddivisa in sottoaree e per ogni sotto area conterrà solo il default delle attività per cui l'utente è abilitato.
Solo le aree e sottoaree che hanno attività abilitato vengono prospettate all'utente

Punti di ingresso

  1. Menu->Amministrazione->Default utente

Prerequisiti

  • Avere le abilitazioni ad operare per poter applicare i default

Modulo di acecsso ai default di area 

Salvataggio dei default di area avvenuto correttamente




Nell'esempio prospettato abbiamo due sottoaree:
  1. Interrogazione per titolo
  2. Interrogazione per autore
Per ogni sottoarea vengono prospettati i seguenti dati incolonnati:
  1. Il nome del campo al quale viene applicato il default
  2. Il default attribuito dal sistema se l'utente non interviene
  3. L'indicazione di chi fornisce i ldefault di sistema che può essere la biblioteca, il polo o in ultimo caso l'applicativo.
  4. Il default gestito dall'utente
I default gestiti dall'utente possono essere indicati sottoforma di:
  • Campi di input testuali
  • Selezioni singole da liste a tendina
  • Selezioni multiple sempre da liste a tendina

Azioni utente

  • Bottone ''Salva". Consente di salvare la configurazione di default oer l'area in cui si sta operando
  • Bottone "Annulla"  torna alla schermata delle aree

Controlli del sistema

  • Gestione dei default guidata. 
  • Elementi per blocco. Numerico fino ad un valore massimo di 50
